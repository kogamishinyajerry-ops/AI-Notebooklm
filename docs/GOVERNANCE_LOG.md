# COMAC AI-Notebooklm Governance Log

This document serves as the canonical record of architectural decisions and governance audits during the development of the COMAC AI-Notebooklm project.

## Core Constraints Implemented

### Constraint C1: Air-Gap Privacy & Compliance
- **Requirement**: Zero telemetry, zero external API calls at runtime.
- **Enforcement Action**: 
    1. Adopted `ChromaDB` local persistent store.
    2. Embedded `BAAI/bge-large-zh-v1.5` directly into the Docker image via `scripts/pre_download_models.py` during the build sequence.
    3. Transitioned frontend architecture from `Vite/React` to `Vanilla HTML/CSS/JS` to eliminate `npm` and CDN dependencies.

### Constraint C2: Anti-Hallucination & Traceability
- **Requirement**: All facts presented by the AI must be anchored to document evidence.
- **Enforcement Action**:
    1. Implemented high-fidelity `PDFParser` (PyMuPDF) to cache Page and Bounding Box (x0,y0,x1,y1) metrics.
    2. Engineered the `AntiHallucinationGateway` middleware to intercept LLM responses. Any `<citation>` XML tag generated by the LLM is cross-verified against the retrieved RAG context. Ungrounded citations are proactively stripped and marked with system warnings.

### Constraint C3: SSOT Integrity & CI/CD Review
- **Requirement**: All code changes must funnel through a structured Review loop.
- **Enforcement Action**:
    1. Integrated a 5-layer Notion Control Tower (Projects -> Specs -> Phases -> Tasks -> Reviews).
    2. Repelled an early-stage cross-contamination event (ALPH code leak) by executing a surgical excision, Git history rewrite, and force-syncing to a new dedicated `AI-Notebooklm` repository.
    3. Mandated Chief Architect (Opus 4.6) PR Gate Reviews for Phases 1 through 5.

## Phase Timeline

- **Phase 4**: Vanilla UI & Canvas BBox Event Loop
