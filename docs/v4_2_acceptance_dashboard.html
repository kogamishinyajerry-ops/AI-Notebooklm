Root cause

Five test modules (test_rate_limit, test_auth_isolation, test_observability, test_llm_health, test_studio) conditionally install a stub core.retrieval.retriever guarded by if "core.retrieval.retriever" not in sys.modules. The stub replaces RetrieverEngine with a _FakeRetrieverEngine that lacks _rrf_fuse. The full-suite alphabetical ordering happened to import the real module upstream — the 8-file subset broke that accident.

Fix

One conftest-level pre-import loop covers 7 fragile-guard modules (retriever, graph_extractor, gateway, prompts, vllm_client, source, studio_output). Stubbers' guards self-skip. Zero test-file edits.
